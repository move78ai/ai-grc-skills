# Contributing

## Contribution rules
- Keep the repo modular. Do not merge multiple skills into a single super-skill.
- Keep the umbrella folder focused on routing, intake, and catalog references only.
- Use lowercase, hyphen-separated folder names.
- Use `SKILL.md` for instructions and `README.md` for GitHub-facing documentation.
- Keep source-safe content only. Do not add raw PDFs, session handoff dumps, or private working notes.

## Update workflow
1. Update or replace the source skill content in the relevant skill folder.
2. Refresh `README.md` if the purpose or included materials changed.
3. Update `SKILLS-CATALOG.md` and `docs/release-assets.md` when new skills are added.
4. Package zip artifacts only during release preparation.
