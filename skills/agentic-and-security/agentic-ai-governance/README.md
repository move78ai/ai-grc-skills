# Agentic AI Governance

## What this skill does
Workflow-first governance guidance for agent suitability, bounded autonomy, accountability, oversight, and phased rollout.

## Who it is for
- Teams deciding whether an agentic use case should proceed at all.
- Operators defining autonomy boundaries, permissions, and accountability splits.
- Reviewers designing oversight, rollout, and end-user responsibility controls.

## Use this skill when
- You need a go, conditional go, or hold decision for an agentic use case.
- You need autonomy limits, human checkpoints, or owner assignments.
- You need rollout conditions, monitoring expectations, or residual-risk acceptance.

## Typical inputs
- Agent purpose, scope of action, tools, integrations, and user population.
- Autonomy level, write access, approval gates, and escalation constraints.
- Current governance controls, testing posture, and known risk concerns.

## Typical outputs
- Governance profile or use-case suitability memo.
- Bounded-autonomy matrix, accountability split, or oversight plan.
- Rollout decision, monitoring plan, or residual-risk memo.

## How it works
- Start by deciding whether the agent should exist in the proposed form.
- Set explicit limits on autonomy, permissions, identities, and overrides.
- Tie approval to named owners, evidence, and ongoing supervision.
