# MITRE ATLAS AI Security

## What this skill does
Workflow-first MITRE ATLAS support for AI threat models, attack scenarios, mitigations, detections, and exercise planning.

## Who it is for
- Teams using MITRE ATLAS as the primary public threat taxonomy for AI-enabled systems.
- Operators preparing threat model registers, exercises, or detection hypotheses.
- Reviewers who need executive-ready security outputs grounded in concrete attack patterns.

## Use this skill when
- You need an AI threat model based on MITRE ATLAS tactics and patterns.
- You need a red-team or purple-team exercise plan for an AI system.
- You need mitigations, detections, or residual-risk documentation tied to named threats.

## Typical inputs
- System profile, attacker assumptions, deployment context, and trust boundaries.
- Known attack concerns, supplier exposure, detections, and operational dependencies.
- Current mitigations, logging posture, and review objectives.

## Typical outputs
- Threat model register or scenario catalog.
- Detection and monitoring plan or exercise plan.
- Executive risk memo with mitigation priorities.

## How it works
- Select the tactics and attack paths that actually fit the system in scope.
- Turn the threat model into testable detections and response expectations.
- Keep residual-risk decisions explicit when coverage is partial or immature.
