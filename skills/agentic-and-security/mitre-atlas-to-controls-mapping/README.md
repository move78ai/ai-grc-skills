# MITRE ATLAS to Controls Mapping

## What this skill does
Operator-grade control-mapping support that turns MITRE ATLAS findings into control themes, evidence needs, and owner actions.

## Who it is for
- Teams translating ATLAS findings into governance, assurance, or implementation actions.
- Operators who need a defensible bridge from threat scenarios to controls and evidence.
- Reviewers deciding which findings become release conditions, exceptions, or tracked remediation.

## Use this skill when
- You already have ATLAS findings and need control implications.
- You need evidence requests, implementation actions, or control tracker outputs.
- You need an executive or audit view of why the mapped controls matter.

## Typical inputs
- ATLAS findings, threat scenarios, and system context.
- Current control baseline, owner structure, and evidence posture.
- Decision target such as release approval, audit review, or remediation planning.

## Typical outputs
- Control mapping tracker and implementation register.
- Evidence checklist or action plan.
- Residual-risk or executive summary memo.

## How it works
- Translate threat findings into control themes rather than pretending one-to-one equivalence.
- Assign owners and evidence requirements to each mapped action.
- Use the mapping to route into governance, assurance, or release decisions.
