# TEVV / AI Assurance / Pre-Deployment Testing

## What this skill does
Workflow-first TEVV support for assurance programs, test plans, metrics, release gates, and retest triggers.

## Who it is for
- Teams standing up or tightening an AI assurance and TEVV capability.
- Operators preparing release or deployment approval packages for specific systems.
- Reviewers who need go/no-go logic tied to test evidence and residual risk.

## Use this skill when
- You need a TEVV program design or refresh.
- You need a system-specific pre-deployment test and approval pack.
- You need retest triggers, independence checks, or evidence-based release conditions.

## Typical inputs
- System context, intended use, lifecycle stage, and deployment decision in scope.
- Current testing evidence, evaluation metrics, reviewers, and known limitations.
- Dependencies, high-impact use concerns, and post-release monitoring expectations.

## Typical outputs
- TEVV plan, evidence checklist, or metric register.
- Go/no-go memo, residual-risk note, or independent-review log.
- Retest trigger log and post-deployment monitoring actions.

## How it works
- Define the assurance scope before collecting evidence or debating thresholds.
- Keep test coverage, independence, and release authority visible in the same package.
- End with explicit pass criteria, unresolved risks, and next review triggers.
