# AI GRC and Regulatory

## What this skill does
Router skill for AI governance, risk, compliance, security, supplier, and assurance workflows.

## Who it is for
- Leads who need one intake point for governance, legal, security, supplier, and assurance work.
- Operators who need to split a messy request into the right workstreams before drafting artifacts.
- Teams that want one consistent output style across multiple AI governance domains.

## Use this skill when
- The user does not yet know which skill family applies.
- The task spans several domains, such as regulation plus supplier risk or incident plus board reporting.
- You need a single scoping memo before handing work to narrower skills.

## Typical inputs
- A short description of the system, use case, or decision in scope.
- Known jurisdictions, frameworks, suppliers, or stakeholders already involved.
- Current blockers, missing facts, or evidence gaps.

## Typical outputs
- A routed problem statement with the right skill family or families.
- A split work plan with owners, evidence needs, and decision labels.
- A merged operator-ready output when several child skills need to be combined.

## How it works
- Classify the object in scope before suggesting any framework, law, or control path.
- Split multi-domain workstreams instead of collapsing them into one blurred answer.
- Return outputs in one consistent operator format with explicit owners and next reviews.
