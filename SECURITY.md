# Security Policy

## Scope
This repository is a public source library of AI governance and compliance skill instructions. It is not a production SaaS service, it does not host customer environments, and it should not contain private operating data.

## Report a vulnerability privately
Do not open a public issue for a suspected security problem.

Use GitHub Private Vulnerability Reporting for this repository:

- [Report a vulnerability privately](https://github.com/move78ai/ai-grc-skills/security/advisories/new)

If that path is unavailable, contact the repository maintainer through the repository owner profile and request a private reporting channel before sharing details.

## What to include
- A short description of the issue and why it matters
- The affected file, workflow, dependency, or repository path
- Clear reproduction steps or proof of concept
- Any known impact, exposure conditions, or required permissions
- Your suggested remediation, if you have one

## Disclosure expectations
- Do not disclose the issue publicly before triage is complete.
- Give the maintainer a reasonable opportunity to confirm scope and prepare a fix.
- If the issue turns out to be out of scope for this source repository, the maintainer may redirect you to a more appropriate reporting path.
